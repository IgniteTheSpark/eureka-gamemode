# Eureka 设计简报

> 版本：v1.0 | 2026-05-24
> 用途：给设计师 / 设计工具的输入文档。包含 Eureka 设计需要的全部产品 / 交互 / 视觉约束,不含技术架构噪音。
> 来源：从 [Phase A 产品定义](phase-a-product-definition.md) 与 [Phase B 架构蓝图](phase-b-architecture-blueprint.md) 抽出对设计有意义的部分。
> 期望产出：`DESIGN.md` + design tokens(JSON / CSS variables)+ 可选视觉预览。

## 一、产品一句话

**Eureka 是一个个人 AI 助手,硬件语音是它的线下输入入口。**

用户对着硬件说话(会议、对话、闪念),app 转成结构化记录;一个统一 AI 助手能跟你过去的全部记录对话。

## 二、品牌定位与气质

| 维度 | 描述 |
|---|---|
| 核心情感 | 「私人 AI 在为我工作」—— 沉静、有秩序、值得信任 |
| 与竞品视觉差异 | 不要走「热闹 / 多色 / 卡通」的笔记 app 风;也不要走「冷峻 / 极简 / 黑白」的工程师风。落在「**克制的精致**」—— 像一个有审美的智能伙伴 |
| voice & tone(微文案) | 简洁、自然、不过度热情;不卖萌、不用感叹号堆砌、不喊口号 |
| 关键词 | 沉稳 / 流畅 / 克制 / 私人感 / 值得托付 |

## 三、目标用户(两个人群,一套视觉)

| Persona | 画像 | 主用场景 |
|---|---|---|
| 知识沉淀型 | 学生 / 创作者 / 白领 | 把闪念、想法、笔记、待办沉淀下来,以后能找回、能问 AI |
| 商务型 | 商务人士 / 老板 | 日程管理(会议、约见)、人脉记录(名片)、把交谈整理为可执行项 |

**关键约束**:不是两个 app、不是两套视觉。一套设计系统通吃两种人群,差异**只在「主屏」**(用户切换「日历为主」或「资产为主」)。

视觉气质要同时满足:
- 学生 / 创作者用着不觉得「老气、商务、压抑」
- 商务用户用着不觉得「学生气、轻浮、不专业」

→ 落点是**中性专业 + 一点温度**。

## 四、核心交互场景(设计要覆盖的)

### 4.1 AI 对话(产品核心)
- 用户输入文字或语音 → agent 流式回复
- agent 回复可能含:纯文本 / 资产卡片(自动创建的)/「沉淀为资产」入口(无明确意图时)
- 多轮对话气泡(user 右 / agent 左,或同侧错位)
- 流式 token 出现需要视觉节奏(打字感,但不卡顿)

### 4.2 闪念捕捉
- 用户对着硬件说话(demo 期用浏览器麦克风模拟)
- 录音中状态:有视觉反馈(波形 / 圆点呼吸 / 计时)
- 转录完成 → 显示转录文本 + 自动派生的资产卡片(可能多张)
- 文字输入路径:输入框,无录音视觉

### 4.3 时间流
- 按日期分组(今天 / 昨天 / 前天 ... / 周末等)
- 每天内按时间倒序
- 类型筛选 tab(全部 / 待办 / 事件 / 想法 / 名片 / 记账)
- 点日期可定位 / 跳转到日历

### 4.4 日历(差异化头牌之一)
- 月视图为主(可能也有周 / 日视图)
- 日格子上同时显示「事件 + 待办」(两种气质要区分)
- 当前日 / 选中日的强调
- 日格子内 event 的紧凑样式 + 「+N more」

### 4.5 资产库
- 按类型组织(待办 / 事件 / 想法 / 名片 / 记账)
- 待办按状态分组(进行中 / 已完成)
- 记账有简单汇总条(本月 / 本周)
- 名片按名字字母或最近联系排序

### 4.6 资产详情 / 编辑
- 单条资产查看全部字段
- 字段可编辑(inline / form)
- 关联信息:从哪个 transcript 派生(可点回看)

### 4.7 加 skill 流程(差异化头牌之二)
- 用户描述想加的 skill(如「我想记录跑步训练」)
- AI(design agent)流式输出:字段建议 + 视觉建议 + 预览卡片
- 用户微调(icon / 颜色 / 字段顺序)
- 确认 → 新 skill 立即可用

这个流程要有「魔法时刻」感 —— AI 一步步帮你画卡片,你看着它生成。是 demo 的发光点之一。

## 五、最终页面集

5 个核心页面 + 1 个开关:

| 页面 | 说明 |
|---|---|
| AI 对话 | 产品核心。统一助手 |
| 时间流 | 类型筛选 + 点日期定位 |
| 日历 | 显性 feature。月视图为主 |
| 资产库 | 按类型组织 |
| 资产详情 | 单条编辑 |

**呈现模式开关**(用户可切):
- 「日历为主」→ 打开 app 落到日历页(商务人群偏好)
- 「资产为主」→ 打开 app 落到时间流或资产库(知识人群偏好)
- AI 对话在任意模式都常驻可达

## 六、Design Tokens(关键约束)

### 6.1 色板需求

需要定义以下色,带**亮色 + 暗色双模式**的 hex:

| 类别 | 用途 |
|---|---|
| **品牌主色** | 1-2 个,代表 Eureka 的核心识别 |
| **中性色** 6-8 级 | 文本(主 / 次 / 辅)、背景、边框、分割线 |
| **语义色** | success / warning / error / info |
| **7 个 accent 色** | **render_spec 里要用,枚举值固定**:见下 |

**⚠️ 7 个 accent 色是硬约束**(用于资产卡片的视觉区分):

```
accent.blue       默认(如待办 todo)
accent.amber      警示 / 提醒类
accent.green      正向 / 财务类(如记账 expense)
accent.red        紧急 / 已逾期
accent.purple     事件类(如 event)
accent.gray       次要
accent.neutral    无特殊语义
```

设计师可以**重新命名**(比如改成 `accent.primary / accent.warm / accent.success ...`),但**必须保持 7 个语义槽位**,且每个槽位有:
- 主色(用于 icon / 强调)
- 弱色(用于背景 / chip)
- 文字色(用于 accent 上的文字)
- 暗色模式映射

### 6.2 字体系统

- 字体家族:中文 + 西文各 1-2 个,需中英混排好看
- 尺寸 scale:建议 12 / 13 / 14 / 16 / 20 / 24 / 32(供设计师调整)
- 字重:regular / medium / semibold;不用 light(屏上易糊)
- 行高:body 用 1.5,heading 用 1.3 上下

### 6.3 间距 scale

建议 4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 八级,token 命名 `spacing.xs / sm / md / lg / xl / 2xl / 3xl / 4xl`(可由设计师定)。

### 6.4 圆角 / 阴影 / 边框

- 圆角:`radius.sm / md / lg / xl / full`(具体值待定)
- 阴影:`shadow.sm / md / lg`,建议低调,不用大模糊大偏移
- 边框:`border.subtle / default / strong`

### 6.5 运动 tokens

- transition 时长:`duration.fast` (~150ms) / `normal` (~250ms) / `slow` (~400ms)
- 缓动:`easing.in-out`(默认)/ `easing.out`(进入)/ `easing.in`(退出)
- 流式 token 出现的节奏(per-token 间隔 ~30-50ms,或按字符)

## 七、组件视觉规范(设计要产出的)

### 7.1 SkillCard(资产卡片)—— **最重要**

所有资产用统一 `<SkillCard>` 组件渲染,行为由 render_spec 控制。

**支持 4 种 layout**(设计要为每种出样式):

| layout | 用途 |
|---|---|
| `horizontal` | 图标 + 主标题(同行)+ 次标题(下方或同行右),适合 todo / expense / contact |
| `stacked` | 图标 + 主标题(上)+ 多行内容(下),适合 idea / note |
| `inline` | 极紧凑,一行内全部信息,适合时间流密集场景 |
| `compact` | 卡片缩小版,用于日历格子内 |

**SkillCard 的固定结构**(设计师必须按此设计,不能改结构,只能改具体样式):
- icon(emoji 或图标)
- accent 颜色应用(icon 色 / 强调线 / 背景 tint 其一)
- primary_field(主标题文字)
- secondary_field(副标题,可格式化)
- meta_fields(0-3 个 chip / 小字段)
- actions(0-4 个图标动作:check / edit / delete / open)

设计师要为每个 layout 给出:亮 + 暗,默认 + hover + active + selected 状态。

### 7.2 Chat 消息

- user 气泡(右 / 主色背景 / 白字),也可以同侧错位现代风
- agent 气泡(左 / 透明或浅底,跟整体融合)
- 流式 token 出现的动画(打字感,光标闪烁 or 不要光标)
- agent 气泡内嵌资产卡片(SkillCard 的紧凑变体)
- 「沉淀为资产」按钮(在 agent 文字回复下方,不抢主视觉)

### 7.3 闪念捕捉视觉

- 录音中:波形 / 呼吸 dot / 计时器(选一种,要克制)
- 转录文本气泡(跟 user 消息区分,这是「机器代我说话」)
- 转录后的资产卡片(同 SkillCard 系统)

### 7.4 时间流

- 日期分隔条(粘性 sticky?今天用强调色?)
- 卡片之间的紧凑节奏(不要每张都浮起来,会很乱)
- 类型筛选 tab 在顶部

### 7.5 日历

- 月视图主样式
- 格子内 event chip(用 accent.purple 系)和 todo chip(用 accent.blue 系)
- 当前日强调(圆 / 实心 / 边框,选一种)
- 选中日强调(不同于当前日)
- 「+N more」溢出处理
- 顶部:月份切换 + 模式切换(月 / 周 / 日)

### 7.6 资产库

- 类型分区(tab 或 sidebar)
- 待办的状态分组样式(已完成的视觉降级)
- 记账汇总条(简单的本月总额 + 分类条形)

### 7.7 输入区

- 文字输入框 + 麦克风按钮 + 发送按钮
- FAB(浮动操作按钮)用于全局闪念入口

### 7.8 Modal / Drawer / Wizard

- Add Skill Wizard(多步,带预览,流式接受 AI 建议)

### 7.9 顶栏 / 导航 / 模式切换

- 顶栏简洁
- 模式切换(日历为主 / 资产为主)放哪儿:设置里?顶栏右上?设计师决定

## 八、视觉「魔法时刻」候选(差异化体验,设计要重点表达)

这些是 demo 时给人留下印象的关键瞬间,值得设计师在视觉上「加点东西」:

1. **闪念语音录入** → 转录文字流出来 → 资产卡片**逐张飘进 / 弹出**(不是同时刷出来)。让人看到 AI 「在帮我整理」。
2. **统一助手流式回复** + 工具调用时**资产卡片就地生成**。让人看到 AI 「在帮我做事」。
3. **Add Skill Wizard**:AI 流式给字段建议 + 视觉建议 + 实时预览卡片刷新。让人感到「AI 帮我设计了一个属于我的功能」。
4. **呈现模式切换**:从「资产为主」切到「日历为主」时,有一个视觉过渡(slide / fade / 重组),不要硬切。

## 九、不在设计范围内(本轮不做)

- 真实硬件 UI(按键、灯效、屏)
- 会议详情页(数据模型留位,本轮不实现)
- 多用户 / 账号体系 / 登录注册流程
- 营销官网 / 落地页
- 邮件 / 推送通知样式
- Onboarding 引导流程(可后续做)

## 十一、产物回到开发后会怎么用

为了你跟设计工具沟通时心里有底,以下是产物的下游使用:

- **色板 → render_spec 的 enum 值**:Phase B 现在写的是 `accent_color: 'blue' | 'amber' | ...`,你的设计如果改成 `'primary' | 'warm' | ...`,前端就用你的命名,enum 跟着改。
- **tokens → CSS variables / Tailwind**:Phase D 前端会把 tokens 文件直接接入构建。
- **组件规范 → React 组件**:每个组件按你的视觉规范写。
- **DESIGN.md → 长期视觉权威**:以后加新组件、新页面都参考它。

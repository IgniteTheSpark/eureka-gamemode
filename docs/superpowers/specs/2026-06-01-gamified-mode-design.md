# Eureka Assistant — 游戏化模式设计 (Gamified Mode) v2

- **Date:** 2026-06-01
- **Repo:** `eureka-gamemode` (standalone, forked from Eureka-Assistant)
- **Status:** **v2 — 取消小镇 / 地图,转向「三视图 + mascot 双轨成长」。** 不再需要定位 / GPS(v1 的地点模型作废,§11 隐私阻塞点随之消失)。collector 规则(EXP 曲线、多样性阈值、"完成"判定)作为独立一轮**后定**。下一步:整体框架 / 展示形态 → 实现计划。
- **v1 → v2 变更:** 小镇 / 地图 / 一切定位 **全部砍掉**(太复杂、暂无好思路);engagement 主引擎从"录入→世界(地图)生长"换成"录入→**喂养 mascot**"。三视图(今日 / 时间 / 资产)大体复用 app 现有结构(日历 + 图鉴),不重建。

---

## 1. 概述 / 定位

把助理本质包进一层**轻**游戏皮:

> 一只代表 agent 的 **mascot** 陪着你。你随口的**闪念**被它接住、结晶成有类型的资产 —— 这等于在 **feed 它**。它靠两件事成长:你 feed 的**内容多样性**(解锁新形态)+ 你**把事情做完 / 合成**(给 EXP 升级)。整个 app 就三个视图:**今日 / 时间 / 资产**。

录入 → 沉淀 → 回溯 的核心链路不变。

**定位:** 这是一个 **C 端个人助理**产品 —— 硬件(BLE 闪念卡)+ 自然语言输入,帮各类用户(老到小)记录日常该记的东西。**会议录音那类是另一个产品 / 市场(toB / toBtoC / to 老板),明确不在本产品范围**(长音频已 strip)。

**核心难点 = 让用户养成"随手记"的习惯**,所以用**宠物 + 任务**来 engage(受多邻国启发:游戏化引导完成 + 积分 / 攀比促留存)。

**engagement 哲学(已拍板:混合 / opt-in):默认温柔**(无连胜 / 无愧疚 / 无排行榜 —— 延续 §4 护栏);**竞争元素(连胜 / 排行榜 / 攀比 / 宠物差异化攀比)作为可选开关,且依赖社交 / 多用户,后置**。默认值定义产品气质:温柔、广谱、可持续。

非目标:**不**做"宠物本身即目的"的养成游戏(宠物永远服务记录);**默认不**做强制 / 愧疚机制(竞争元素仅 opt-in 且后置);游戏层永远服务"记录与回忆"。**取消小镇 / 地图与一切定位**。

## 2. 范围与平台

- **双端**:
  - **网页(先做)** — 登录后的全量游戏化控制台。
  - **App(原生,后做)** — BLE 硬件**闪念捕捉口**(按键即录、离线补传)。app 的价值是**随身、零摩擦的捕捉入口**,**不再**以"拿定位"为独有价值。
- **无地理位置**:不取定位、不存经纬、不画地图。
- **先单用户**:在现有单用户后端(`USER_ID=default`)跑通;账号 / 登录 / 多用户后置。

## 3. 三视图(app 骨架)

使用与回溯拆成三个视图,mascot 贯穿其中:

### 3.1 今日(主页 / mascot 之家)

首页是**混合展示**,三块:

1. **今日待完成(agent 每日现场生成)** —— 一个 agent 每天生成一次的清单(读你的未完成项 + 习惯 + 技能);它同时是 mascot 的 **EXP 闭环面**:勾掉 / 完成一项 = 闭环 = EXP(详见 §5)。
2. **今天的闪念 turns** —— 今天 flash session 的 input turns,按时间排开。
3. **随时自由对话** —— 内嵌 ask-agent 聊天,首页直接开聊。

**mascot 活在这里**:长按 mascot / 闪念键捕捉(输入先出现 → 思考 → 结晶 → mascot 接住,闭环则庆祝 + EXP);它的**心情 / 活跃度**(= 等级的外在表现)也在此体现。

> **今日待完成怎么来(已拍板:agent 每日现场生成)。** 一个 **daily 任务生成 agent**(结构化输出,与现有 flash dispatcher 同形)每天跑一遍,读未完成 todo、今日日程、记账 / idea 等习惯与技能,**当场**生成今日清单。默认**每天生成一次**(当天首次进首页 / 早晨定时),**缓存当天、可手动刷新**;生成中先显示昨日结转 / 占位。能学习惯、会变 —— 代价是一次真实 agent 调用(见 §11)。

### 3.2 时间维度

- 按时间陈列的资产 / 日程(用 `effective_at`):往回翻是记忆,往前是日程 / 待办。≈ 现有**日历(流 / 月 / 年)换皮**。

### 3.3 资产维度(图鉴)

- 资产按类型陈列成收藏品;**多样性**在这里一眼可见(你追踪了多少种东西)。含「**+ 新收藏**」加技能向导(复用现有 `AddSkillWizard`,后端 designer/confirm/dispatcher 照搬)。

## 4. Mascot(agent 的具象)+ 双轨成长

- **三合一**:agent 的脸 + 闪念接球手 + 聊天人格。
- **状态动画**:闲逛 / 听(长按时,复用现有「正在聆听」)/ 思考(处理中)/ 庆祝(沉淀 / 闭环)/ 夜里睡。
- **双轨成长(已拍板):**
  - **① 多样性 → 解锁形态(广度)**:你 feed 的**资产种类**越多(图鉴越宽),解锁 mascot 的**新形态 / 主题外观**。鼓励"什么都记一点"。
  - **② 完成 → EXP → 等级(深度)**:"**录入后完成**"的闭环事件给 **EXP**,EXP 升级 → 影响 mascot 的**心情 / 活跃度 / 光环**。鼓励"把事做完、做合成"。
- **护栏(延续 v1 既定):**
  - **不做强制每日任务** —— EXP 机会性触发、**只加不罚**,久不来不掉级、不愧疚。
  - 两轨产出**只有外观 / 心情**,**绝不锁功能**(核心动作永远可用)。
- **气质:默认温柔 / 竞争 opt-in(已拍板,见 §1)** —— 上面两轨(多样性解锁、完成 EXP)是**温柔内核**,**MVP 只做这个**。**连胜 / 排行榜 / 攀比 / 宠物差异化攀比**是**可选竞争层**,依赖多用户 / 社交,**后置**。

## 5. 今日待完成 = EXP 闭环面(agent 生成)

首页的「今日待完成」清单由 agent **每日现场生成**(§3.1),它同时是 mascot 的 **EXP 来源**:完成一项闭环 → EXP(深度轨,§4)。**裸录入不给 EXP**(只喂多样性轨)。

> **硬原则:任务必须 grounded 在用户自己的捕捉 / 需求 / 技能,绝不凭空造。** 例:闪念"后天提醒我参加饭局"→ 任务"确认参加"(勾选 todo);用户建了记账 → 任务"每日准点记三餐";用户记了跑步计划 → 任务"每日跑步记录 + 心情"。生成器只把用户**已表达的需求**翻译成今日可勾选的闭环,不发明用户没说过的事。

agent 生成的每一项带一种**完成形态**:

- **勾选(check)** —— todo、日程:点一下标记完成。
- **每日定额 + 时段(daily slots)** —— 记账:早 / 午 / 晚三餐各一笔、落在对应时段(项上显示 1/3 → 3/3,**时间点匹配**)。
- **阈值 → 升华(synthesis)** —— idea:数量到阈值,agent 提示"合并成一条更深的想法",做了即完成。
- **agent 抽象(custom)** —— 任意自定义技能:agent 推断它"完成一个闭环"长什么样。

完成判定:直给项(todo / 日程)勾选即落库;推断项(三餐完整度、idea 升华、自定义闭环)由生成项自带的**完成判定**对数据求值。完成 → 触发 EXP。

> 每项的完成判定细则、各给多少 EXP、升级曲线、生成频率与质量调参 —— 都属 **collector 规则**,作为独立一轮后定,本期框架不依赖精确数值。

## 6. 交互逻辑

- **长按 mascot / 闪念键** = 捕捉(镜像硬件键)。节奏复用已实现的"输入先出现、再分析":听 → 松开 → 思考(火花)→ 资产**结晶** → mascot 接住;**若闭环**则庆祝 + EXP。
- **短按 mascot** = 随时对话(**ask-agent**)。
- **点资产** → 展开**会话卡**:这条闪念的 input/output 会话流,可**继续追问**(**ask-asset**,上下文绑定)。
- **三视图切换**:今日 / 时间 / 资产。

## 7. 资产沉淀与回溯

- **沉淀**:闪念 → 现有 flash pipeline(dispatcher → skills → assets)→ typed asset,带 `effective_at`,**不带 location**。
- **回溯三条路**:翻**时间维度**、翻**资产维度(图鉴)**、问 **mascot**("这个月花了多少 / 上次关于 X 说了啥" → query + HTML 报告)。

## 8. 界面映射(现有 → 新)

| 现有 | 新模式 |
|---|---|
| 日历(流 / 月 / 年) | **时间维度**(换皮,保留) |

| Library(skills grid + 资产) | **资产维度 / 图鉴** + 「+ 新收藏」加技能 |

## 9. 数据模型变更

- **已移除 files**(phase-1 strip 完成;`InputTurn.text` 保留为原始闪念)。
- **无 `location` / `places` / 亲和表**(随地图取消)。
- **新增 mascot 状态**:`level`, `exp`, `unlocked_forms`(多样性解锁集), `diversity_kinds`(已 feed 的 skill 集合);`mood` / `activity` 可派生。
- **新增 EXP 事件**:闭环事件日志(`type`, asset/skill 引用, `exp`, `ts`)。可建表或从现有数据派生。
- **新增 daily plan(今日待完成)**:agent 每日生成的清单,缓存当天 —— `{ day, items: [{ type, title, 完成形态, 完成判定, exp, status }] }`。一天一行(或派生)。
- `effective_at` 保留(驱动 时间维度 + 到期提醒)。
- **Auth**:本期不动(单用户)。

## 10. 网页 MVP 范围(先做这些)

## 11. 风险与待定

- **daily 生成的成本 / 延迟 / 缓存**:每日待完成是一次真实 agent 调用。默认**每天生成一次 + 缓存当天 + 手动刷新**,生成中显示昨日结转 / 占位;别在每次进首页都重算。
- **生成清单的质量与稳定性**:每天不一样可能让人无所适从或质量飘。需要稳定的"基底项"(未完成 todo / 今日日程直给)+ 有限的"推断项"(三餐 / idea 升华 / 自定义),而非满屏新花样。
- **"完成"判定**:什么算"完整一天的消费"?idea 合并阈值?各事件 EXP 与升级曲线?→ 属 **collector 规则**,后定,不阻塞框架。
- **EXP 平衡**:既要奖励到位、又不能沦为刷数据 / 逼迫感。**只加不罚 + 无强制任务**是底线。
- **美术成本**:mascot 多状态 + 多形态(多样性解锁)= 真实美术量;MVP 控形态数,先少量。
- **滩头用户(待定)**:"老到小都需要"是愿景,不是滩头。第一批最痛的用户是谁 → 决定 onboarding、默认任务集、以及温柔/竞争的默认微调。值得在做深之前定一个 beachhead。
- **范围**:按 §10 分期,先把 **今日 + mascot + 三视图 + 加技能** 在网页单用户上跑通。

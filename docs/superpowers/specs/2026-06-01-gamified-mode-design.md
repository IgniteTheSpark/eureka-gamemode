# Eureka Assistant — 游戏化模式设计 (Gamified Mode)

- **Date:** 2026-06-01
- **Branch:** `gamified-mode`
- **Status:** Design approved in brainstorm; pending user spec review → then implementation plan.

---

## 1. 概述 / 定位

把 Eureka 的"助理本质"包进一层有空间感的游戏皮:

> 一只代表 agent 的 **mascot**,陪你在一座**专属像素小镇**里生活。你随口的**闪念**被它当场接住、结晶成有类型的资产,钉在"你当时**在哪、什么时候说的**"那个点上。回溯 = 逛小镇 / 翻 mascot 的任务簿 / 问 mascot。

> 一条贯穿全篇的区分:**捕捉**(你几点、在哪说出这句话——上图、进小镇)≠ **主题**(这句话*讲的是*几点该做什么——进 mascot 任务簿)。地点只记捕捉地点;时间两者都留,各归各的面。

录入 → 沉淀 → 回溯 的核心链路不变,只是从"列表/时间轴"换成"地图 + 角色"。

非目标:不做成宠物养成游戏;游戏层永远服务于"记录与回忆"这件正事。

## 2. 范围与平台

- **双端**:
  - **App(原生,后做)** — 轻量;核心独有价值 = **唯一能拿到用户真实定位/位置**的入口(蓝牙连卡片 + GPS + 离线缓存补传)。把"闪念 + 位置"喂给共享后端。
  - **网页(先做)** — 登录后的全量游戏化控制台:小镇 / mascot / pin 会话 / 图鉴。
- **先做网页端。** 在网页先做时,定位用**浏览器 Geolocation + 地图/地理编码 SDK**(模糊位置 + 地点类型,前台、需授权)——是**真定位、只是粗**,不是假数据。等 app 上来再换成它的精确/后台定位。
- **先单用户、登录后置**:在现有单用户后端(`USER_ID=default`)上把整套跑通;账号 / 登录 / 多用户隔离等接 app、要上线托管时再加。

## 3. 核心:地点模型(推断的相对位置)

地点**不是用户随便摆的**,是从真实物理位置推断出来的相对布局:

1. **取模糊位置**:闪念触发那刻,从地图 SDK 拿一个模糊经纬度(不追求精确)。
2. **推断地点类型**:反向地理编码 → 地点类型(家 / 公司 / 咖啡馆 / 餐厅 / 公园 / 健身 / 在路上 …)→ 决定这个据点渲染成什么风格。
3. **聚类去重**:同一个真实地点反复触发 = **同一个据点**(按距离阈值聚类),不重复造点。
4. **相对布局**:新据点的摆位由它与**上一个触发点的相对方位(bearing) + 空间距离**决定,归一化/抽象后画到小镇上。于是小镇拓扑**真实反映你的移动关系**(谁在谁的哪个方向、大概多远),但:
   - **不暴露绝对位置、真实街道、或两点间的路径/任何中间内容。**
   - 渲染为像素 / 漫画风,看不出真实地图。
5. **兜底**:无法定位(离线补传、未授权、SDK 失败)→ 落到通用「**在路上 / 漫游**」区。

> 网页 MVP:浏览器 Geolocation 给粗经纬度 + 一个 geocoding SDK 给类型;聚类 + 相对布局算法在前端/后端实现。数据稀疏时用少量**播种据点**兜底,避免空镇。

> **只记录捕捉地点,不解析主题地点(已拍板)。** 闪念**内容里**提到的地点(「在冯总那边开会」「去机场」)**不**交给 agent 反推/标注——准确率太低,硬标只会污染地图。地图上的位置永远 = **你说出这句话时人在哪**(捕捉据点)。主题地点若重要,只作为**卡片内容文字**保留,不上图。

## 4. Mascot(agent 的具象)

- 角色三合一:**agent 的脸** + **闪念接球手** + **聊天人格**。
- 状态动画:闲逛 / 听(长按时)/ 思考(处理中)/ 庆祝(沉淀成功)/ 夜里睡。
- **轻养成**:随"持续记录 / 完成待办"解锁表情、小装扮、心情。**奖励的是助理行为本身**,不做等级/属性/RPG。
- 复用现有「正在聆听」语义:长按时 mascot 竖耳朵 = 当前 listening overlay 的换皮。

## 5. 展示逻辑:三个面,各管一件事

回溯不是一个视图扛所有,而是拆成三个面:**小镇=何时何地**、**任务簿=主题时间**、**图鉴=我攒了什么**。问 mascot 横跨三者。

### 5.1 小镇(地图)= 捕捉记忆(在哪 + 何时**捕捉**)

- 一座**持久**的像素小镇;据点见 §3。
- **时间是图层、地点是底图**:
  - 默认看「**今天**」——今天走过的据点连成路线 + 今天落的 pin 高亮。
  - 拖一个**日期滑块**回溯任意一天,那天的路线 + pin 浮现。
  - 缩小 = 多天连成的大地图(取代 月/年 视图)。
- **关键:滑块/图层用的是「捕捉时间」**(你几点开口),**不是**主题时间。小镇 = "我哪天去过哪、在那儿冒了什么念头"的**生活记忆**。一条「下午 3–5 点开会」的闪念,在地图上钉在**你上午说它时**的那个据点、那个时刻;它讲的是下午,只是卡片内容。
- **pin** = 一条闪念沉淀出的资产,图标 = 资产类型(沿用现有 typed card 体系)。

### 5.2 Mascot 任务簿 / 接下来议程 = 主题时间(`effective_at`)

地图是捕捉记忆,撑不起"这件事**几点该发生**"。这一职责交给 mascot 手里的**任务簿**(已拍板形态):

- 一个**按天分组的纵向议程列表**,像 RPG 的 quest log,mascot 拿着。
- 只收**带主题时间**的资产(待办 / 日程);note / idea 这类无 `effective_at` 的不进议程,留在小镇 + 图鉴。
- 到点 mascot **主动 nudge**(复用现有通知 / SSE)。
- 点一条 → **跳回它的捕捉 pin**(地图上点亮),回到"我当时在哪说的"。
- 刻意做轻、与游戏世界同调,**不还原老日历网格**(后置才考虑可展开成月历)。

### 5.3 图鉴 = 我攒了什么(按类型)

资产按类型陈列成收藏品;详见 §8。`小镇=何时何地`,`任务簿=何时该做`,`图鉴=我攒了什么`。

### 5.4 加技能 = 在图鉴里「开新收藏」

技能(skill)= 图鉴里的**一类收藏**(一个收藏册)。"加技能" = **解锁一页新图鉴** + **让 mascot 学会接这类闪念**。

- **入口与形态(已拍板):图鉴里点「+ 新收藏」,走结构化向导**(复用现有 `AddSkillWizard` 换皮成"开新收藏"语言):
  1. 描述你想收集什么(例:跑步训练 / 读书笔记 / 每天喝水量)。
  2. (可选)反问 1–3 个澄清问题(现 design agent 的 clarifier)。
  3. design agent 结构化生成 `payload_schema` + `render_spec`。
  4. 内嵌**收藏品预览卡**(live SkillCard),可改显示名 / 图标 / 配色。
  5. 确认 → 写 `UserSkill`,图鉴多一页。
- **后端整套照搬,不重写**:designer(Gemini structured output)→ `POST /api/skills/confirm` → flash dispatcher **立刻**把新技能注入分类 prompt,**下一条闪念**就能自动归到新类型并按 `render_spec` 出卡。游戏化层只做**换皮 + 改入口/视觉**,不动这条管线。
- **不被养成门禁**:开新收藏是**核心动作,永远可用**;§4 的轻养成只给装扮 / 表情,绝不锁功能。
- **后置(叠加项,非 MVP)**:mascot **主动提议**——当它发现你反复把同类闪念塞进 misc/note,主动问"要不要建个『X』收藏?"。需先做模式识别,叠在向导之上,MVP 后加。

### 5.5 世界生长(engagement 主引擎)

怎么 engage 用户持续用?**主引擎已拍板:录入 → 世界生长。** 被奖励的动作 = 说闪念 = 核心动作本身;产物 = 一座**私人像素小镇 + 记忆档案**。生长必须是你**捕捉历史的诚实渲染**,不是凭空点数。

- **生长模型:纯积累、只增不减(已拍板)。** 小镇**永不退化、不昏暗、不掉等级**,是永久记忆档案 → **零愧疚**。明确不做衰减/回访压力机制。
- **据点成长体系:基础样式 + 进化 + 稀有度(已拍板)。** 由该据点上录的闪念的 **种类 × 数量 × 与地点属性的匹配度** 共同驱动;**不**用闪念内容(内容噪声大、会重复冗余),只用结构化信号。
  - **① 基础样式**:据点按反向地理编码出的**地点类型**(家 / 公司 / 咖啡馆 / 餐厅 / 公园 / 健身 / …)给一个**基础建筑标识**。无法定位 → 通用「在路上 / 漫游」基础标记。
  - **② 进化**:持续录入把基础样式**喂养成**逐级进化的形态(分几级、长什么样 = 留给视觉设计,后续微调)。驱动 = 数量(规模)+ 种类多样度 + **匹配度**(在对的地点录对的类型 → 进化得更点题、更出彩)。
  - **③ 稀有度标识**:每个据点带一个**稀有度徽章 / 边框**(普通 → 精致 → 稀有 → 传奇)。**建议(可调):稀有度主要奖励"匹配质量 + 里程碑",而非单纯堆量** —— 一个长期"在状态"使用的据点,比同等数量但杂乱的据点更快升稀有度。这是**收集 / 炫耀的钩子**。
  - **只增不减**:进化级与稀有度**只升不降**,久不来不退化(对齐零愧疚)。
  - (后置)图鉴填充里程碑(1 / 10 / 50)、路线变密、小镇变宽。
- **地点属性 ↔ 资产类型 亲和表**:餐厅 ↔ 开销 / 美食、健身 ↔ 运动、公司 ↔ 待办 / 会议、公园·咖啡馆 ↔ 想法 / 读书 …;自定义技能(§5.4)创建时**带一个"属于哪些地点"的亲和标签**(可推断 + 可手填)。匹配 = **纯加分**:不匹配的闪念永远照常计数、照常让据点长大,只是不点题,**绝不惩罚或浪费**(核心动作不可被罚;重复 / 冗余闪念也照常计数)。
- **可分享的闪念地图(新增目标)**:各据点进化 + 稀有度组合 → 一张**连贯、完整、可对外分享**的个人闪念地图。分享守 §3 脱敏底线:**只导出抽象像素小镇快照,不含绝对位置 / 真实街道 / 任何原始内容**。
- **里程碑时刻**:据点进化 / 升稀有度 / 收藏到节点 → mascot 庆祝(复用 §6 庆祝动画)。奖励时刻都挂在**真实阈值**上,不乱给。
- **回访拉力来自哪**:地图只增不减、本身不制造回访压力 → **回访靠**① 图鉴集齐进度 ② 轻回溯仪式(每周回顾 / 去年今天,复用现有 HTML 报告)。即:**A(世界生长)是脊梁管"满足感",B(回溯仪式)轻包在外管"再回来"。**
- **冷启动**:早期阈值压低 + §3 播种据点,**第一条闪念就长出肉眼可见的东西**,别让新用户面对空镇。
- **护栏(对齐 §1/§4 非目标)**:不做硬连胜 / XP / 等级 / 每日登录奖励 / 变量奖励上瘾环;§4 轻养成只解锁装扮表情,绝不锁功能。

## 6. 交互逻辑

- **长按 mascot / 闪念键** = 闪念输入(镜像硬件按键)。节奏(复用已实现的"输入先出现、再分析"):
  竖耳朵听 → 松开 → 思考(火花漂浮)→ 资产**结晶**、mascot 接住并钉到当前据点(庆祝)。火花 = 你的原始输入,结晶 = agent 整理结果。
- **短按 mascot** = 随时和 agent 对话(**ask-agent**,保留)。
- **点 pin** → 展开**会话卡**:里面是这条闪念的 **input/output 会话流**(你的原始闪念 = input,agent 的整理/结果 = output,像一小段对话),并可从这里**继续追问**(**ask-asset**,保留,上下文绑定)。
- **翻任务簿** = 点 mascot 身上的议程入口(quest-log 图标/上拉),看带主题时间的待办/日程按天排开;点一条跳回它的捕捉 pin(见 §5.2)。
- 平移/缩放小镇 = 空间+时间回溯;「回到现在」跳回 mascot/今天。

## 7. 资产沉淀与回溯

- **沉淀**:闪念 → 现有 flash pipeline(dispatcher → skills → assets)→ typed asset,**新增**带上 `location`(据点 + 类型 + 相对位置 + 来源,= **捕捉地点**)与 `effective_at`(主题时间)。
- **两套锚定,互不打架**:
  - **小镇 = 捕捉锚定**:位置 = 捕捉据点,时间轴 = 捕捉时间。纯生活记忆。
  - **任务簿 + 问 mascot = 内容锚定**:用 `effective_at` / 金额 / 类型,回答"几点该做 / 这个月花了多少"。
- **回溯/查找的三条入口**:
  1. **逛小镇** — 滑到某天、点某据点,看你在那里钉了什么(取代现有 library 浏览)。*捕捉锚定。*
  2. **翻任务簿** — 看带主题时间的待办/日程按天排开,到点被提醒。*前瞻、内容锚定(见 §5.2)。*
  3. **问 mascot** — "这个月花了多少 / 上次关于 X 说了啥" → 走现有 query + HTML 报告,并能在小镇上**点亮对应捕捉 pin**(mascot/小人回访)。*内容锚定。*

## 8. 界面映射(现有 → 新)

| 现有 | 新模式 |
|---|---|
| 日历(流/月/年)+ 最近 | 一拆为二:**回溯导航**(何时何地)→ **小镇** + 时间滑块/缩放;**主题时间/日程**(几点该做)→ **Mascot 任务簿/议程** |
| Library(skills grid + 资产) | **图鉴 / 收藏**:资产按类型陈列成收藏品;skills 仍在 = "你追踪哪些东西"。`小镇=何时何地`,`任务簿=何时该做`,`图鉴=我攒了什么` |
| 加技能(AddSkillWizard) | 图鉴里的「**+ 新收藏**」向导(换皮,后端照搬);见 §5.4 |
| Chat 入口 | 收进 **mascot**(短按对话) |
| 通知 / 提醒 | 收进 mascot(它帮你递通知 + 任务簿到点 nudge);轻换皮 |
| **文件 实体 / 入口 / 长音频上传** | **彻底去掉**。闪念音频转成文本+资产后**不留存为可浏览文件** |

## 9. 数据模型变更

- **资产新增 `location`**:`{ place_id, place_type, rel_bearing, rel_distance, source: gps|deferred|manual|unknown }`,= **捕捉地点**(`captured_at` 那刻人在哪),不存主题地点。
- **`effective_at`(主题时间)保留并承担新职责**:它是**任务簿/议程的排序键**与**问 mascot 的内容锚定**,**不**参与小镇地图的图层(地图用捕捉时间)。区分清楚:`captured_at`(何时说) vs `effective_at`(何时该发生)。
- **新增 `places` 概念**:GPS 聚类出的据点(`id`, 推断 `type`, 显示名, 相对坐标)。可建表或从 asset.location 派生。据点还需记 **成长状态**:按资产类型的计数(`type_counts`)、`evolution_stage`、`rarity`、`match_score`(派生或缓存),用于 §5.5 的基础样式 + 进化 + 稀有度。
- **地点 ↔ 类型 亲和表**:内建一张 `place_type × asset_type → 匹配` 表;`UserSkill` 加一个 `place_affinity`(自定义技能创建时推断/手填,见 §5.4),供匹配度计算。
- **pin = session**:复用现有 `sessions` + `messages` + `input_turns`;asset 已有 `session_id` / `source_input_turn_id`,会话卡直接读这条 session 的 input/output。
- **移除 files**:删 `files` 表 / `/api/files` / 文件 tile / 文件来源 / 长音频上传路径。
- **Auth**:本期不动(单用户 `USER_ID=default`);多用户/登录后置。

## 10. 网页 MVP 范围(先做这些)

1. 移除 文件 概念(数据 + UI)。
2. 资产加 `location` 字段 + `places`(聚类 + 类型推断 + 相对布局),浏览器 Geolocation + geocoding SDK 取粗位置;稀疏时播种据点。
3. **小镇主视图**:据点 + 今日路线 + pin + 日期滑块(捕捉时间)。
4. **Mascot**:状态动画 + 长按闪念(结晶节奏)+ 短按对话。
5. **Mascot 任务簿/议程**:按天分组的待办/日程列表(`effective_at` 排序)+ 到点 nudge + 点条目跳回捕捉 pin。
6. **pin 会话卡**:展开看 input/output + 继续追问(复用现有 chat 渲染)。
7. **图鉴**:Library 换皮为按类型的资产收藏 + 「**+ 新收藏**」加技能向导(换皮 `AddSkillWizard`,后端 designer/confirm/dispatcher 照搬,见 §5.4)。
8. **世界生长反馈(engagement 主引擎,见 §5.5)**:据点 **基础样式 + 进化 + 稀有度**(数量 × 种类 × 匹配度驱动,只增不减)+ 内建亲和表 + 里程碑触发 mascot 庆祝。
9. 像素 / 漫画风视觉层(设计语言)。

后置:原生 app(真 GPS / 蓝牙 / 离线补传)、登录 / 多用户、mascot 养成深度、多天大地图、任务簿可展开成月历、世界生长进阶(路线变密/小镇变宽)、**可分享地图快照导出**、自定义技能亲和手填 UI、轻回溯仪式(每周回顾/去年今天)、**mascot 主动提议建收藏**、隐私脱敏控制。

## 11. 风险与待定

- **隐私**:真 GPS 即便风格化展示,底层是敏感数据。需定存储/脱敏策略(只存据点+相对位置、不存原始经纬?本地优先?opt-in?)。— **待定,实现前必须拍板**。
- **数据稀疏**:只在家↔公司两点会显得空 → 播种据点 + 「在路上」+ 时间层补丰富度。
- **美术成本(随据点进化体系放大)**:像素 mascot(多状态动画)+ 各地点类型 ×(基础样式 + 多级进化形态)+ 稀有度边框 = 真实且不小的美术量。MVP 控规模:**少数地点类型 × 2–3 级进化 + 一层廉价稀有度边框/光效**(叠加层而非整张重绘);更多形态后续设计。
